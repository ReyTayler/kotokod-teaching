## Суть в 30 секунд

**Раньше:** админ создавал учётку, система генерила временный пароль, админ передавал его вручную (небезопасно). Учётка создавалась с пассивным включением 2FA по ролям.

**Теперь:**
1. Админ создаёт учётку → система генерит одноразовую invite-ссылку (48 ч)
2. Админ передаёт ссылку сотруднику (вручную)
3. Сотрудник переходит по ссылке → сам задаёт пароль
4. Обязательная настройка 2FA (TOTP или email-OTP, выбор на фронте)
5. Успешный вход в систему

**Безопасность:** plaintext-токен не хранится (только SHA-256 в БД), статус сессии проверяется через token_version (смена пароля/2FA/email → инвалидация всех старых сессий).

---

Всё на Django 5.1 + DRF. Без зависимостей (bcrypt, pyotp, PostgreSQL используются из существующего стека).
