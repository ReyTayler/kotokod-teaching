# Чеклист соответствия 152-ФЗ / ИБ РФ (организационная часть)

**Дата:** 2026-06-06
**Назначение:** организационно-юридические обязанности оператора ПДн, которые **не
реализуются в коде** (или реализуются лишь частично). Технические меры описаны в
`docs/superpowers/specs/2026-06-06-rbac-unified-auth-design.md`, раздел «Соответствие ИБ РФ».

> ⚠️ **Это не юридическая консультация.** Документ составлен разработчиком на основе
> открытых источников и подлежит проверке профильным юристом / ответственным за обработку
> ПДн (DPO). Законодательство меняется (см. поправки 2025–2026).

## Контекст проекта

KOTOKOD обрабатывает ПДн: **детей** (ФИО, дата рождения, телефон, класс), **родителей**
(ФИО, телефон), **преподавателей** (ФИО, телефон, теперь — логин/пароль/2FA). По 152-ФЗ
это делает школу **оператором персональных данных**. Категория ПДн — «иные» (не специальные,
не биометрические). Предполагаемый уровень защищённости ИСПДн — **УЗ-3/УЗ-4** (уточняется
при классификации и определении модели угроз по ПП 1119).

## Чеклист

### 1. Регистрация и уведомления
- [ ] Подать **уведомление в Роскомнадзор** об обработке ПДн (внесение в реестр операторов).
- [ ] Регламент **уведомления РКН об утечке**: первичное — в течение **24 часов** с момента
      выявления; итоги внутреннего расследования — в течение **72 часов**.

### 2. Согласия (с 01.09.2025 — отдельный документ)
- [ ] Согласие на обработку ПДн оформлено **отдельным документом**, а не пунктом оферты.
- [ ] Отдельные согласия под **каждую цель** обработки.
- [ ] Для **детей** — письменное согласие **родителя/законного представителя** (ст. 64 СК РФ);
      14–18 лет — ребёнок с одобрения представителя.
- [x] Факт согласия фиксируется в системе (`students.consent_*`) — технический крючок (миграция 015) **и UI реализованы**. *Организационная часть (форма/текст согласия отдельным документом) — выше, остаётся `[ ]`.*

### 3. Документы оператора
- [ ] **Политика обработки ПДн** разработана и **опубликована** (доступна субъектам).
- [ ] Приказ о назначении **ответственного за организацию обработки ПДн**.
- [ ] Перечень обрабатываемых ПДн, целей, сроков хранения; правила доступа сотрудников.
- [ ] Согласие сотрудников (преподавателей) на обработку их ПДн.

### 4. Защищённость ИСПДн (ПП 1119 + Приказ ФСТЭК №21)
- [ ] **Классификация ИСПДн** и определение уровня защищённости (УЗ).
- [ ] **Модель угроз** безопасности ПДн.
- [x] **Технические меры реализованы в коде** (см. спеку: ИАФ/УПД/РСБ): **ИАФ** — 2FA (TOTP+email-OTP),
      bcrypt, lockout, rate-limit на вход; **УПД** — RBAC (роли teacher/manager/admin, `requireRole`);
      **РСБ** — `security_audit_log`. *TLS/бэкапы — на этапе деплоя (см. `deploy-runbook.md`), остаются как прод-задача.*
- [ ] Хостинг и БД — **на территории РФ** (Beget VPS) → локализация 242-ФЗ. *(организационно-деплойное)*
- [ ] Исключить передачу ПДн граждан РФ во внешние сервисы вне РФ (**выпил Google Sheets —
      Phase 5**, см. ROADMAP). *Технически — ⏳ Phase 5 в работе.*
- [x] Email-2FA использует **российский SMTP — Beget** (`SMTP_*` в `.env`, `services/mailer.js`).
      Письмо с кодом не содержит ПДн. *Завести боевой ящик вместо тестового — остаётся (организационное).*

### 5. Реагирование и контроль
- [ ] Регламент реагирования на инциденты ИБ (кто, что, сроки).
- [ ] Периодический просмотр `security_audit_log` (подозрительные входы, блокировки).
- [ ] Расходы на ИБ ≥ 0,1% годового дохода (условие смягчения оборотного штрафа при утечке).

## Риски при несоблюдении (с 30.05.2025, 420-ФЗ)
- Утечка ПДн (повторная) — **оборотный штраф 1–3% выручки**, от 25 до 500 млн ₽.
- Неуведомление РКН об утечке — **1–3 млн ₽**.
- Обработка без согласия / нарушение требований — отдельные штрафы (ужесточены).
