## Общие соглашения для всех backfill-скриптов

Шаблон ровно такой же, как в прототипе:

1. **Чистая функция парсинга** `extractX(rows, ...deps) → Array<entity>` — без I/O.
2. **Loader** `loadX()` — переиспользует `services/sheets.js`.
3. **main**: `--dry-run`, `--help`, упсёрт через `ON CONFLICT DO UPDATE … WHERE … IS DISTINCT FROM …`, `RETURNING (xmax = 0) AS inserted`.
4. **JSON-итог** в stdout, прогресс — в stderr.
5. **Exit code** 0/1.
6. **Идемпотентность** — двойной прогон даёт 0 изменений.
7. **`pool.end()`** в `finally`, чтобы скрипт корректно завершался.

**Контекст:** имя группы в Sheets выглядит как `Scratch ср 17:00`, `Python вт 19.00 / пт 17:30`, `Roblox ИНДИВ пн 18-00`. Один или несколько слотов через `/` или пробелы. Регулярка из CLAUDE.md: `(день_недели)[^0-9]*(чч)[:.-](мм)`.

- [ ] **Step 1: тесты**

- [ ] **Step 2: Реализация**

День недели в JS getDay-конвенции: 0=вс, 1=пн, 2=вт, 3=ср, 4=чт, 5=пт, 6=сб.
Маппинг по русским сокращениям, нижний регистр, первые 2 буквы достаточно (но точное совпадение надёжнее: `пн|вт|ср|чт|пт|сб|вс`).

```js
const DAY_MAP = { 'вс': 0, 'пн': 1, 'вт': 2, 'ср': 3, 'чт': 4, 'пт': 5, 'сб': 6 };

function parseTimeSlots(groupName) {
  if (!groupName) return [];
  const re = /(вс|пн|вт|ср|чт|пт|сб)[^0-9]*(\d{1,2})[:.\-](\d{2})/gi;
  const slots = [];
  for (const m of groupName.matchAll(re)) {
    const day = DAY_MAP[m[1].toLowerCase()];
    const hh = String(m[2]).padStart(2, '0');
    const mm = m[3];
    slots.push({ day_of_week: day, start_time: `${hh}:${mm}:00` });
  }
  return slots;
}

function parseLessonDuration(groupName) {
  return /\b45\s*минут/i.test(groupName) ? 45 : 90;
}

module.exports = { parseTimeSlots, parseLessonDuration };
```

```sql
WITH d AS (SELECT id FROM directions WHERE name = $2),
     t AS (SELECT id FROM teachers   WHERE name = $3)
INSERT INTO groups (name, direction_id, teacher_id, is_individual,
                    lesson_duration_minutes, lessons_per_week, vk_chat)
SELECT $1, d.id, t.id, $4, $5, $6, NULLIF($7, '') FROM d, t
ON CONFLICT (name) DO UPDATE SET

    teacher_id              = EXCLUDED.teacher_id,
```

### Task 7: `lessons` + `lesson_attendance` backfill

**Files:**
- Create: `scripts/backfill-lessons.js`
- Create: `scripts/backfill-lessons.test.js`

**Источник:** листы «Журнал группы» и «Журнал индивы» в журнальной таблице. Каждая строка:
- A: date
- B: teacher
- C: group
- D: lesson_number
- E: student name
- F: 'Был' / 'Не был' → present
- G: fixedAt (используется как `submitted_at`, опционально)
- H: token → `submitted_by_token`
- I: record_url
- J: lesson_type label (Замена/Перенос/'') — маппится в lesson_type
- K: originalTeacher (если Замена)

**Группировка:** строки с одинаковыми `(date, group, lesson_number, token)` — это **один** lesson, а student/present каждая — это `lesson_attendance`.

**lesson_type:**
- `J === 'Замена'` → `'substitution'`
- `J === 'Перенос'` → `'reschedule'`
- иначе → `'regular'`

**lesson_duration_minutes:** берётся из связанной `groups.lesson_duration_minutes` (selectом после group lookup).

**Тест extractLessons:**
```js
test('extractLessons: группировка строк в lesson + attendance', () => {
  const rows = [
    ['2024-09-04', 'Иванов', 'Scratch ср 17:00', '1', 'Аня', 'Был',  '04.09 18:00', 'tok1', '', '', ''],
    ['2024-09-04', 'Иванов', 'Scratch ср 17:00', '1', 'Боря', 'Не был', '04.09 18:00', 'tok1', '', '', ''],
    ['2024-09-11', 'Иванов', 'Scratch ср 17:00', '2', 'Аня', 'Был',  '11.09 18:00', 'tok1', 'http://rec', 'Замена', 'Петрова'],
  ];
  const { lessons, attendance } = extractLessons(rows, /* sheetName */ 'Журнал группы');
  // Two lesson records, three attendance records
  assert.strictEqual(lessons.length, 2);
  assert.strictEqual(attendance.length, 3);
  assert.deepStrictEqual(lessons[0], {
    lesson_date: '2024-09-04',
    teacher_name: 'Иванов',
    group_name: 'Scratch ср 17:00',
    lesson_number: 1,
    submitted_by_token: 'tok1',
    record_url: null,
    lesson_type: 'regular',
    original_teacher_name: null,
  });
  assert.deepStrictEqual(lessons[1].lesson_type, 'substitution');
  assert.deepStrictEqual(lessons[1].original_teacher_name, 'Петрова');
});
```

**Upsert lessons** — на каждый lesson:
```sql
WITH g AS (SELECT id, lesson_duration_minutes FROM groups WHERE name = $3),
     t AS (SELECT id FROM teachers WHERE name = $2),
     ot AS (SELECT id FROM teachers WHERE name = $8)   -- nullable
INSERT INTO lessons
  (lesson_date, teacher_id, group_id, lesson_number,
   lesson_duration_minutes, lesson_type, record_url,
   submitted_by_token, original_teacher_id)
SELECT $1, t.id, g.id, $4, g.lesson_duration_minutes, $5, NULLIF($6,''), $7, ot.id
FROM g, t LEFT JOIN ot ON TRUE
ON CONFLICT (lesson_date, group_id, lesson_number, submitted_by_token) DO NOTHING
RETURNING id, (xmax = 0) AS inserted
```

**lesson_attendance** — после lesson insert/lookup:
```sql
WITH s AS (SELECT id FROM students WHERE full_name = $2)
INSERT INTO lesson_attendance (lesson_id, student_id, present)
SELECT $1, s.id, $3 FROM s
ON CONFLICT (lesson_id, student_id) DO NOTHING
```

**Acceptance:**
- count(lessons) = количество уникальных (date, group, lesson_number, token) троек из обоих листов.
- count(lesson_attendance) = количество строк в обоих листах (за вычетом тех, где `student` не разрезолвился).
- Случайная выборка 5 уроков сверяется глазами с Sheets.

```sql
  SELECT id, teacher_id FROM lessons
  WHERE lesson_date = $1 AND group_id = (SELECT id FROM groups WHERE name=$2)
    AND lesson_number = $3 AND submitted_by_token = $4
)
INSERT INTO payroll (lesson_id, teacher_id, total_students, present_count, payment, penalty)
SELECT l.id, l.teacher_id, $5, $6, $7, $8 FROM l
ON CONFLICT (lesson_id) DO UPDATE SET
    total_students = EXCLUDED.total_students,
    present_count  = EXCLUDED.present_count,
    payment        = EXCLUDED.payment,
    penalty        = EXCLUDED.penalty
WHERE
    payroll.total_students IS DISTINCT FROM EXCLUDED.total_students
 OR payroll.present_count  IS DISTINCT FROM EXCLUDED.present_count
 OR payroll.payment        IS DISTINCT FROM EXCLUDED.payment
 OR payroll.penalty        IS DISTINCT FROM EXCLUDED.penalty
RETURNING (xmax = 0) AS inserted
```

**Acceptance:** count(payroll) ≤ count(lessons); сумма payment'ов совпадает с суммой колонки G в Sheets.

### Task 9: Orchestrator `backfill-all.js`

**Files:**
- Create: `scripts/backfill-all.js`

Запускает все 7 скриптов по порядку в одном Node-процессе (импортирует их `main`), собирает JSON-итоги в один массив, печатает summary в stdout. Останавливается при первой ошибке.

```js
require('dotenv').config();
const steps = [
  ['directions', require('./backfill-directions')],
  ['teachers',   require('./backfill-teachers')],
  ['tokens',     require('./backfill-tokens')],
  ['groups',     require('./backfill-groups')],
  ['students',   require('./backfill-students')],
  ['lessons',    require('./backfill-lessons')],
  ['payroll',    require('./backfill-payroll')],
];

// каждый module экспортирует runBackfill({dryRun}) → result
async function main() {
  const dryRun = process.argv.includes('--dry-run');
  const all = [];
  for (const [name, mod] of steps) {
    process.stderr.write(`\n=== ${name} ===\n`);
    const result = await mod.runBackfill({ dryRun });
    all.push(result);
  }
  process.stdout.write(JSON.stringify(all, null, 2) + '\n');
}
```

**Каждый backfill-скрипт должен экспортировать функцию `runBackfill({dryRun})` →`Promise<result>`**, оставаясь при этом запускаемым `node scripts/backfill-X.js`. Это означает рефакторинг прототипа `backfill-directions.js`: вынести main в `runBackfill`, оставить тонкий entrypoint.
