# Блокировка отметки будущих уроков + фиксация даты в форме отчёта

Дата: 2026-07-13

## Проблема

В Teacher SPA учитель может:
1. Отметить занятие, запланированное на будущую дату (ещё не наступившее), через календарь.
2. В форме отчёта по уроку вручную поменять дату урока на любую другую — вместо той, что реально запланирована.

Нужно заблокировать оба момента на фронте и на бэке (бэк — обязательная защита, фронт может быть обойдён).

## Правило «будущего»

Сравнение только по дню: `occurrence.date > todayMsk()` (MSK, без учёта времени начала урока).
Сегодняшний урок можно отмечать в любое время дня, даже до его начала по расписанию.

## Frontend (`journal_django/frontend/teacher-src/`)

### `pages/calendar/OccurrenceMenu.tsx`
Пункт «Отметить урок» получает три состояния вместо двух:
- `status === 'done' || status === 'cancelled'` → пункт не рендерится (как сейчас).
- `occ.date > todayIso` (будущее) → пункт рендерится, но `disabled`, с видимой подписью под текстом («доступно в день урока») + `title`-тултип. Стиль `.occ-menu-item:disabled` уже есть в `styles/pages.css:77`, нужно добавить только класс подписи (по аналогии с существующим disabled-пунктом «Перейти в чат» этого же файла).
- иначе → активная кнопка, как сейчас.

`todayIso` — через существующие `todayMsk()` / `isoDate()` из `lib/dates.ts`.

### `components/lessons/LessonForm.tsx`
Поле «Дата урока» становится нередактируемым: `DateInput` получает `disabled={true}`.
Проп уже поддерживается общим компонентом `@shared/components/form/DateInput` (admin-src, alias `@shared`) — его трогать нельзя (фаза миграции: admin-src только читается из teacher-src).

Источник даты не меняется — по-прежнему `initialDate` (клик по занятию в календаре) либо сегодняшняя (открытие из «Мои уроки», где список всегда строится только на сегодня). Убирается только ручная правка.

Предупреждение про штраф (`date !== todayIso`, показывается для просроченных занятий) не трогаем — работает как раньше.

**Осознанный побочный эффект (подтверждён пользователем):** учитель больше не может задним числом вписать другую дату отчёта (например, вчерашнюю, если забыл отметить вовремя) — дата всегда жёстко равна дате занятия.

## Backend (`journal_django/apps/teacher_spa/`)

### `services.py::submit_lesson`
Ранняя проверка сразу после распаковки `date` из `validated`, до всех обращений к БД:

```python
if date > format_msk_date():
    return {'success': False, 'error': 'Урок ещё не наступил — отметить его можно только в день занятия или позже.'}
```

`format_msk_date()` уже импортирована и используется в этой функции для расчёта штрафа (строка ~174) — переиспользуем, новый импорт не нужен. Сравнение строк `'YYYY-MM-DD'` корректно лексикографически для ISO-дат.

Формат ошибки — `{'success': False, 'error': ...}` (тот же паттерн, что «Группа не найдена» и другие бизнес-ошибки в этой функции), а НЕ через `serializers.ValidationError` в `SubmitLessonSerializer`. Причина: `custom_exception_handler` (`apps/core/exceptions.py`) переписывает тело ValidationError в `{"error": "Validation failed", "details": {...}}` — фронт (`ApiError.message`) увидел бы только generic «Validation failed», а не конкретный текст. `success: false` идёт прямо в `result.error` → `setSubmitError` → `<div className="lf-error">`, что уже показывает содержательное сообщение.

Это defense-in-depth: обычный учитель до этой ветки не дойдёт (кнопка на фронте скрыта/задизейблена), проверка подстраховывает от гонки состояний (устаревший кэш календаря) и прямых запросов к API.

## Тесты
- Backend: новый кейс в `journal_django/apps/teacher_spa/tests/test_teacher_spa_api.py` — `submitLesson` с датой в будущем → `{'success': False, 'error': ...}`, без записи в БД (lesson/attendance/payroll не создаются).
- Frontend: ручная проверка в браузере (автотестов React-компонентов в teacher-src пока нет — не добавляем новый тест-раннер ради одной проверки).

## Вне охвата
- Учительский SPA `MyLessonsPage` не может открыть форму для будущей даты в принципе (список строится только на сегодня) — изменений там не требуется.
- Admin SPA не затронут — там свой флоу редактирования уроков.
