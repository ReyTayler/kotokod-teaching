# Teacher SPA: «Мои группы» + календарь — дизайн (2026-07-12)

Утверждено пользователем в диалоге 2026-07-12.

## 1. «Мои группы»

- Вкладки «Мои занятия / Замена» удаляются. `useAllData` (`/api/getAllData`) из GroupsPage уходит, но хук ОСТАЁТСЯ: CalendarPage лениво тянет его, когда заменщик отмечает урок ЧУЖОЙ группы из календаря (данных группы нет в /api/getData).
- Карточка группы: без кнопки «Отметить урок», кликабельна → роут `/groups/:group` (encodeURIComponent имени).
- Бейдж «Долг» — честный: `N с долгом`, если у ЛЮБОГО ученика `remaining <= 0` (раньше смотрел только `students[0]`).
- Новая страница `GroupDetailPage`:
  1. **Ученики** — имя, возраст, «осталось N оплаченных уроков» (`remaining` из `/api/getData`); подсветка: красный при ≤0, жёлтый при 1–2.
  2. **Прогресс** — матрица посещаемости, идентичная admin `GroupProgressBlock`; компонент переносится в `shared/` и используется обоими SPA.

## 2. Замена — серверный вывод

- Из `SubmitPayload`/API убираются `isSubstitution`/`originalTeacher`; присланные значения → 400.
- `submit_lesson` определяет сам: группа отправителя → обычный урок; иначе, если в `planned_lessons` есть занятие этой группы на эту дату с `teacher_id` = отправитель (назначено админом «Сменить преподавателя») → урок пишется как `substitution` c `original_teacher_id` = владелец группы; иначе 403.
- Форма данных в БД и «Мои занятия» не меняется.

## 3. Календарь

- Сетка 8–21 уже есть (WeekGrid HOUR_START=8, HOUR_END=21) — проверить клампы граничных занятий.
- `/api/calendar` occurrence: + `durationMinutes` (из `groups.lesson_duration_minutes`), + `vkChat` (для чата из контекстного меню; override-группы не входят в /api/getData). Аддитивно, parity-тесты дополнить.
- WeekGrid: высота блока = `duration/60 × ROW_H − 4` (фолбэк 60 мин); кластеризация пересекающихся занятий → дорожки с равной шириной (как на референс-скриншоте).
- Визуальные статусы: `done` — приглушённый блок; `cancelled` — перечёркнутый текст (+ приглушение).
- Линия «сейчас» в колонке сегодняшнего дня + автоскролл к текущему часу при открытии.
- Клик по ячейке в teacher SPA → контекстное меню у точки клика:
  - «Отметить урок» → `LessonForm` с предзаполненной датой занятия (скрыт для done/cancelled);
  - «Открыть карточку группы» → `/groups/:group`;
  - «Перейти в чат группы» → `occ.vkChat` (disabled, если нет);
  - «Подробности» → существующий `LessonPopup`.
  - Admin-поведение (сразу popup c операциями плана) не меняется.

## 4. Новый эндпоинт

`GET /api/group-progress?group=<name>` (teacher_spa, `IsTeacher`): имя → group_id, доступ = владелец группы ИЛИ назначен хотя бы на одно `planned_lesson` этой группы; ответ = `apps/groups/services.get_group_progress()` (тот же контракт, что admin `/api/admin/groups/:id/progress`).

## Дополнение 2026-07-13: тип урока выводит сервер

- Из LessonForm убран сегмент «По расписанию / Перенос»; `lessonType` удалён из payload и отклоняется API с 400 (тот же паттерн, что isSubstitution).
- `submit_lesson` выводит `lesson_type='reschedule'` сам: плановая строка группы/даты/препода перенесена НА эту дату (`moved_from_date` задан, не cancelled) — `repository.planned_lesson_is_moved`. Приоритет: substitution > reschedule > regular.
- Поле «Ссылка на запись» → блок «Запись урока»: иконка-скрепка, placeholder, кнопка очистки, зелёная рамка при валидном URL, мягкое предупреждение (не блокирует), если текст не похож на ссылку.
