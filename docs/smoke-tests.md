# Smoke tests (ручной чеклист)

Прогоняется перед мерджем каждой фазы миграции и после крупных рефакторингов.
Сервер локально: `npm start` → http://localhost:3000.

Авторизация: для прохождения чеклиста нужен валидный teacher-токен.
Если токенов нет — посмотреть на листе «Токены» в таблице журнала.

---

## Авторизация

- [X] /api/validateToken с **валидным** токеном → `{valid: true, teacher: '...'}`
- [X] /api/validateToken с **невалидным** токеном → `{valid: false, error: 'Неверный токен'}`
- [X] /api/validateToken с **пустым** body → ошибка валидации

## Чтение данных

- [X] /api/getData с валидным токеном → возвращает группы только этого препода
- [X] /api/getAllData (для замен) → возвращает все группы всех преподов
- [X] /api/refreshData → сбрасывает кеш, читает заново, возвращает данные препода

## Отправка урока (submitLesson)

- [X] Обычный урок (90 минут, 3 ученика, 2 присутствуют) → success, payment > 0, lessonNum = previous + 1
- [X] 45-минутный урок (в названии группы есть «45 минут») → шаг счётчика 0.5, оплата по тарифу halfLesson
- [X] Замена (isSubstitution=true, originalTeacher указан) → запись в журнал с колонками «Замена» / «За кого»

## Отчёт и расписание

- [X] /api/report → таблица текущей недели, статусы done/pending/overdue корректны
- [X] /api/report/refresh → сбрасывает кеш и редиректит на /api/report
- [X] /api/schedule → все группы со временами, отсортированные по дню+времени
- [X] /api/schedule/refresh → сбрасывает кеш и редиректит на /api/schedule

## SPA (public/Index.html)

- [X] Открыть http://localhost:3000 → загружается интерфейс
- [X] Ввести валидный токен → попадаем на экран со списком групп
- [X] Выбрать группу → видны ученики, можно отметить присутствие
- [X] Отправить урок → success-сообщение, счётчики обновились в таблице Sheets

---

## Когда чеклист красный

Если хотя бы один пункт упал — **не мерджить** фазу, разбираться. До Phase 3 (cutover) откат прост: вернуть импорт `const sheets = require('./services/sheets')` и заменить `repo.` обратно на `sheets.`.
