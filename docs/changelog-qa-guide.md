# Журнал изменений — гайд по ручной проверке в браузере

Дата: 2026-07-06 (v2 — под новый дизайн: описания, статусы, откат из строки,
модалка вместо отдельной страницы). Раздел: **/admin/changelog**, только admin.

Автоматика прогнана (751 pytest + e2e-смоук по API); этот документ — про
браузер: вёрстка, UX, полный набор пользовательских сценариев.

---

## 0. Подготовка

```powershell
cd C:\Users\ilyap\TestKOTOKOD\journal_django
.venv\Scripts\python.exe manage.py runserver        # http://localhost:8000
```

- Бандл admin SPA пересобран — в браузере **Ctrl+F5**.
- Нужны аккаунты: **admin** (вы) и **manager** (для RBAC-проверок).
- Тестовый мусор чистится самой фичей — откатом своих операций. Для тестовых
  записей используйте префикс, например `__тест__`.
- Журнал знает историю только с момента установки триггеров (сегодняшней
  миграции) — прошлые правки в нём не появятся.

## 2. Лента: описания и статусы по каждому типу действий

Делайте действие → обновляйте ленту → проверяйте строку: **Время** (06.07 11:11,
полная дата — в title при наведении), **Действие** (иконка + короткая метка),
**Описание** (человекочитаемое, с именами групп/учеников), **Кто** («Имя (роль)»),
**Статус** «применено».

### 2.1 Планирование (самые важные описания)

| # | Действие | Ожидаемое «Действие» и «Описание» |
|---|---|---|
| 2.1.1 | Перенести занятие плана | «Перенос урока» / `Перенос ПИ1012 №20: 2026-07-09 12:00 → 2026-07-07 12:00` (имя вашей группы, реальные даты) |
| 2.1.2 | Отменить занятие | «Отмена урока» / `Отмена занятия {группа} №N (дата время)` |
| 2.1.3 | Доп. занятие | «Доп. занятие» / `Доп. занятие {группа}: дата время` |
| 2.1.4 | Сменить преподавателя занятия | «Замена преподавателя» / `Смена преподавателя {группа} №N: {старый} → {новый}` (имена, не id) |
| 2.1.5 | Сгенерировать план группы | «Генерация плана» / `План {группа}: создано занятий — N` |
| 2.1.6 | Постоянная смена расписания | «Смена расписания (план)» / `Смена расписания {группа}: занятий изменено — N` |

### 2.2 Уроки и посещаемость

| # | Действие | Описание |
|---|---|---|
| 2.2.1 | Учитель проводит урок (teacher SPA) | «Проведение урока» / `Проведён урок {группа} №N (дата): отмечено X из Y`; Кто = **имя преподавателя (преподаватель)** |
| 2.2.2 | Правка клетки посещаемости в админке | «Правка посещаемости» / `Посещаемость: {ученик} → был/не был` |
| 2.2.3 | Удалить урок | «Удаление урока» / `Удалён урок {группа} №N (дата)` |

### 2.3 Ученики, оплаты, справочники

| # | Действие | Описание |
|---|---|---|
| 2.3.1 | Зачислить ученика в группу | «Зачисление» / `Зачисление: {ученик} → {группа}` |
| 2.3.2 | Отчислить (удалить членство) | «Отчисление» / `Отчисление: {ученик} из {группа}` |
| 2.3.3 | Внести оплату | «Оплата» / `Оплата {сумма} ₽: {ученик}` |
| 2.3.4 | Удалить оплату | «Удаление оплаты» / `Удалена оплата {сумма} ₽: {ученик}` |
| 2.3.5 | Создать направление/преподавателя/ученика/скидку | «Новое направление» и т.п. / `Направление «имя»: создание` |
| 2.3.6 | Изменить справочник | «Правка …» / `… «имя»: изменено (поля)` |
| 2.3.7 | Удалить справочник (это архив) | «… в архив» / `… «имя»: в архив` |
| 2.3.8 | Правка начисления в «Зарплате» | «Правка начисления» |
| 2.3.9 | Учётки: создать/сброс пароля/2FA | «Новая учётка»/«Сброс пароля»/«Сброс 2FA»; описание с email; **в diff нет** `password`/`twofa_secret` |

### 2.4 Не должно попадать в ленту

- Входы/выходы/просмотры (GET) — они в «Журнале ИБ».
- Консольные backfill-команды — данные фиксируются, но строки-операции в ленте
  не будет (осознанное v1-ограничение).

## 4. Модалка деталей

| # | Шаги | Ожидаемо |
|---|---|---|
| 4.1 | Клик по строке ленты | Открывается **модальное окно** (не отдельная страница): заголовок — метка действия; внутри — описание, время, email, `METHOD /api/...`, uuid операции |
| 4.2 | Операция-изменение | Секция сущности → бейдж «изменение» (синий) → таблица «Поле / Было / Стало» — только изменившиеся поля |
| 4.3 | Операция-создание / удаление | Бейдж зелёный/красный; снапшот значений («—» на пустой стороне) |
| 4.4 | Составная операция (проведение урока) | Секции: Урок / Посещаемость (N записей) / Членство / Начисление |
| 4.5 | Массовая (генерация плана) | Секция «Плановое занятие — N записей» свёрнута, кнопка «Показать» |
| 4.6 | Ссылки «Группа #id», «Ученик #id» | Ведут в карточку раздела (модалка закрывается); у удалённых строк и сущностей без своих страниц — просто текст |
| 4.7 | Длинное значение (>200 символов) | Усечено + «показать полностью» |
| 4.8 | Кнопки футера | «Закрыть» всегда; «Откатить операцию» — только admin и только если операция откатываемая |
| 4.9 | Открыть операцию с учётками | Баннер «…операции с учётками не откатываются», кнопки отката нет |
| 4.10 | Тёмная тема | Модалка, diff, бейджи, иконки читаемы |

## 5. Откат — успех

| # | Сценарий | Шаги | Ожидаемо |
|---|---|---|---|
| 5.1 | Из строки ленты | «Откатить» у операции-изменения → confirm-модалка (метка, описание, чипы, «Будет отменено изменений: N») → «Откатить» | Toast «Операция отменена…». Данные вернулись. Строка получила статус **«откачено»**, кнопка «Откатить» у неё исчезла; сверху — новая строка «Откат» (синий бейдж) / `Откат операции: отменено изменений — N` |
| 5.2 | Из модалки деталей | Открыть операцию → «Откатить операцию» | Тот же confirm-флоу |
| 5.3 | Откат создания | Создать направление → откатить | Запись исчезла из раздела; новая запись после этого создаётся без ошибок (sequence) |
| 5.4 | Откат удаления | Удалить оплату → откатить | Оплата вернулась (тот же id/сумма), баланс восстановился |
| 5.5 | Откат составной | Откатить «Проведение урока» | Исчезли урок/посещаемость/начисление, счётчики уроков вернулись |
| 5.6 | Redo запрещён (изменение от 2026-07-07) | Открыть строку «Откат» / уже откаченную операцию | У строки «Откат» и у откаченной операции кнопки «Откатить» НЕТ (`revertable=false`); в деталях — причина «операции отката не откатываются» / «она уже откачена». Прямой `POST …/revert` → 400 «Откат операции отката недоступен.» / «Операция уже откачена.» |
| 5.7 | Журнал ИБ | После отката | Событие `changelog_revert` с вашим email |
| 5.8 | Обновление разделов | Раздел с данными открыт во второй вкладке | После отката и возврата на вкладку данные обновились |

## 6. Откат — отказы

| # | Сценарий | Шаги | Ожидаемо |
|---|---|---|---|
| 6.1 | Конфликт | Изменить направление (А), изменить ещё раз (Б) → откатить **А** | Confirm-модалка переключается в «Откат невозможен: конфликт»: `Направление #id — запись изменена позже этой операции (name)` + ссылка **«история записи»** → лента с чипом-фильтром по этой записи. Данные не тронуты |
| 6.2 | Разрешение | Откатить Б, затем А | Оба проходят, значение исходное |
| 6.3 | Двойной клик | Быстро дважды «Откатить» в confirm | Кнопка блокируется («Откатываем…»), дублей нет |
| 6.4 | Уже откачено | У строки «откачено» | Кнопки «Откатить» нет ни в ленте, ни в модалке («операция уже откачена») |
| 6.5 | Учётки | Операции account.* | Кнопки нет (лента и модалка), статус обычный «применено» |

## 8. Если нашли баг

1. **UUID операции** (в шапке модалки, серым) — скопировать;
2. шаги воспроизведения (какое действие в каком разделе);
3. скриншот + DevTools Console (ошибки) и Network (упавший запрос);
4. роль, под которой работали.

Тестовые откаты безопасно откатывать обратно (5.6) — журнал ведёт полный след.
